Tema 1

Antecedentes, definición y clasificación de bases de datos

Las bases de datos surgen por la necesidad de almacenar, organizar y consultar información de forma eficiente. Antes de los sistemas actuales, muchas instituciones guardaban datos en papel, archivos físicos o listas separadas. Con el crecimiento de las empresas, universidades, bancos y sistemas administrativos, fue necesario crear sistemas capaces de guardar grandes volúmenes de datos, evitar duplicidad y permitir consultas rápidas.

Definición sencilla

Una base de datos es una colección organizada de información que se almacena generalmente en un sistema informático para poder registrarla, consultarla, modificarla y protegerla.

Dato

Es un valor individual. Ejemplo: “María López”, “95”, “2026-05-23”.

Información

Es el dato interpretado. Ejemplo: “María López obtuvo 95 puntos en Matemática”.

Base de datos

Es la estructura que permite guardar muchos datos relacionados y consultarlos cuando se necesitan.

Clasificación general

Tipo de base de datos Descripción Ejemplo
Relacional Organiza la información en tablas relacionadas mediante llaves. MySQL, PostgreSQL, SQL Server, Oracle, Access.
No relacional / NoSQL Guarda datos en documentos, claves-valor, grafos o columnas. MongoDB, Redis, Neo4j.
Centralizada Los datos se guardan en un solo servidor o ubicación principal. Sistema académico local de una institución.
Distribuida Los datos se distribuyen en varios servidores o sedes. Sistema bancario con sucursales conectadas.
En la nube La base de datos se aloja en servicios de internet. Amazon RDS, Azure SQL, Google Cloud SQL.

Ejemplos de la vida diaria

Matrícula: estudiantes, carreras, grupos, pagos y asignaturas.
Inventario: productos, entradas, salidas, proveedores y existencias.
Biblioteca: libros, autores, préstamos, estudiantes y fechas de devolución.
Ventas: clientes, facturas, productos, pagos y descuentos.
Bancos: clientes, cuentas, transacciones y préstamos.
Sistemas académicos: docentes, estudiantes, notas, horarios y aulas.

Tema 2

¿Qué es un SGBD o DBMS?

Un SGBD o DBMS es el software que permite crear, administrar y utilizar bases de datos. Funciona como intermediario entre los usuarios, las aplicaciones y los datos almacenados.

Objetivo principal

Permitir que la información se almacene de forma ordenada, segura y accesible, facilitando operaciones como registrar, consultar, actualizar y eliminar datos.

Ejemplos de SGBD

Oracle Database, Microsoft SQL Server, PostgreSQL, MySQL, MariaDB y Microsoft Access.

Herramientas o funciones principales

1
Creación de estructuras:

Permite crear bases de datos, tablas, campos, llaves y relaciones.

2
Manipulación de datos:

Permite insertar, modificar, eliminar y consultar registros.

3
Recuperación de información:

Permite buscar datos mediante consultas, filtros y reportes.

4
Seguridad:

Controla usuarios, permisos y acceso a la información.

5
Copias de seguridad:

Permite respaldar datos para recuperarlos ante fallos.

6
Importación y exportación:

Permite trasladar información desde o hacia Excel, CSV u otros sistemas.

Tema 3

Modelo Entidad-Relación E-R

El modelo Entidad-Relación es una técnica de diseño conceptual que permite representar la información importante de un sistema antes de crear las tablas en una base de datos. Ayuda a responder: ¿qué datos se guardarán?, ¿cómo se relacionan?, ¿cuáles son sus atributos? y ¿qué reglas debe cumplir el sistema?

Entidad

Objeto, persona, lugar o concepto del que se desea guardar información.

Ejemplo: Estudiante, Docente, Producto, Libro.

Atributo

Característica que describe a una entidad.

Ejemplo: nombre, cédula, fecha_nacimiento, precio.

Relación

Asociación entre dos o más entidades.

Ejemplo: Estudiante se matricula en Asignatura.

Tipos de relaciones con ejemplos

1 : 1

Uno a uno

Un registro de una tabla se relaciona con un solo registro de otra tabla.

Ejemplo: una persona tiene un expediente académico único.

Persona 1 ─── 1 Expediente
1 : N

Uno a muchos

Un registro de una tabla se relaciona con varios registros de otra tabla.

Ejemplo: un cliente puede realizar muchas facturas.

Cliente 1 ─── N Facturas
N : M

Muchos a muchos

Varios registros de una tabla se relacionan con varios registros de otra tabla.

Ejemplo: un estudiante puede cursar varias asignaturas y una asignatura puede tener varios estudiantes.

Estudiante N ─── M Asignatura
Importante: una relación muchos a muchos normalmente se resuelve con una tabla intermedia. Por ejemplo: Estudiante, Asignatura y Matrícula.

Elementos mínimos que debe tener un modelo E-R

  • Entidades con nombres claros.
  • Atributos principales de cada entidad.
  • Llave primaria en cada entidad.
  • Relaciones entre entidades.
  • Cardinalidad: 1:1, 1:N o N:M.
  • Tabla intermedia cuando exista relación N:M.

Tema 4

Guía paso a paso para crear una base de datos

01

Analizar el problema

Identifica qué información necesita guardar el sistema. Ejemplo: una biblioteca necesita libros, estudiantes y préstamos.

02

Identificar entidades

Busca sustantivos importantes: Estudiante, Libro, Préstamo, Autor, Categoría.

03

Definir atributos

Agrega datos de cada entidad: nombre, código, fecha, precio, estado, dirección.

04

Establecer llaves

Cada tabla debe tener una llave primaria. Las llaves foráneas conectan tablas relacionadas.

05

Diseñar el modelo E-R

Dibuja entidades, atributos y relaciones con su cardinalidad.

06

Crear la base de datos

Utiliza un SGBD como MySQL, PostgreSQL, SQL Server, Oracle o Access.

Ejemplo básico en SQL

Este ejemplo representa una relación muchos a muchos entre estudiantes y asignaturas, resuelta mediante la tabla intermedia matriculas. 

CREATE DATABASE universidad_demo;

CREATE TABLE estudiantes (
  id_estudiante INT PRIMARY KEY,
  carnet VARCHAR(20) NOT NULL,
  nombres VARCHAR(80) NOT NULL,
  apellidos VARCHAR(80) NOT NULL
);

CREATE TABLE asignaturas (
  id_asignatura INT PRIMARY KEY,
  codigo VARCHAR(20) NOT NULL,
  nombre VARCHAR(100) NOT NULL,
  creditos INT NOT NULL
);

CREATE TABLE matriculas (
  id_matricula INT PRIMARY KEY,
  id_estudiante INT NOT NULL,
  id_asignatura INT NOT NULL,
  fecha_matricula DATE NOT NULL,
  FOREIGN KEY (id_estudiante) REFERENCES estudiantes(id_estudiante),
  FOREIGN KEY (id_asignatura) REFERENCES asignaturas(id_asignatura)
);
Consulta de ejemplo usando JOIN 
SELECT 
  e.carnet,
  e.nombres,
  e.apellidos,
  a.nombre AS asignatura,
  m.fecha_matricula
FROM matriculas m
INNER JOIN estudiantes e ON m.id_estudiante = e.id_estudiante
INNER JOIN asignaturas a ON m.id_asignatura = a.id_asignatura;

Diseño y creación de una base de datos con modelo E-R
